# Contributing

Thanks for helping make this the best list of RL environments on the web!

## Guidelines

- **One environment per entry.** If a repo ships several distinct environments, link the project once.
- **Format:** `- [Name](canonical-url) — One neutral sentence describing what it is.`
- **Keep descriptions factual and short** — a single sentence, no marketing language, no "blazing-fast".
- **Link to the source of truth** (usually the GitHub repo or official docs), not a blog post.
- **Pick the best-fitting section.** If your entry doesn't fit any, propose a new section in your PR.
- **Order entries** roughly by prominence/standard-ness within a section, or alphabetically.
- **No dead or unmaintained projects** unless they remain a genuine standard/benchmark.

## How to add an entry

1. Fork the repo and edit `README.md`.
2. Add your entry under the most relevant heading.
3. Make sure the link works and the badge/anchor table of contents still lines up.
4. Open a pull request describing the environment in one line.

## Suggestions & corrections

Spotted a broken link, a wrong description, or a missing classic? Open an issue — small fixes are just as valuable as new entries.
